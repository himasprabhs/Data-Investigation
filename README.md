# SQL Murder Mystery – Case Study
### Who Killed the CEO?

This project is a SQL-based investigative case study where I solved a fictional murder mystery by analyzing structured data using SQL queries.

The objective was to identify the killer by logically connecting evidence across multiple tables — similar to real-world analytical problem solving.

---

## 🧩 Problem Overview

A CEO has been murdered, and multiple datasets were provided containing crime scene reports, suspect details, witness statements, and activity logs.

The task was to uncover **who committed the murder** using only SQL queries and logical reasoning.

---

## 🛠️ Skills & Concepts Used

- SQL Joins (INNER, LEFT)
- Subqueries
- Common Table Expressions (CTEs)
- Filtering and aggregations
- Analytical and logical reasoning

---

## 📄 Project File

- `Capstone Project - The Murder Mystery - Who Killed the CEO.sql`  
  Contains step-by-step SQL queries with clear comments explaining the investigation process.

---

## 🧠 Key Learnings

- Translating open-ended problems into structured SQL logic
- Using data to validate assumptions
- Writing readable, well-commented SQL
- Thinking beyond queries — like an analyst

---

## 🎯 Use Case

This project demonstrates my ability to analyze complex datasets, apply SQL for decision-making, and communicate insights clearly — aligned with **Data Analyst / Business Analyst roles**.
